# Grok Frontend Scaffolds

Generated by Grok (Super) during Session 9 using WebCoRE screenshots 
and FRONTEND_SPEC.md / WIZARD_SPEC.md as reference.

## Files

index.html      — Piston list + status page
editor.html     — Editor tree with statement wizard modal
wizard.html     — Condition wizard multi-step flow (standalone)
integrated.html — Editor + condition wizard + HA entity fetcher combined
ha-fetcher.js   — Standalone HA entity fetcher (calls HA directly, for testing only)

## Known Issues to Fix Before Building On This

- `end execute`, `end if`, `end with` — semicolons removed but may have crept 
  back in some files. Verify and remove any trailing semicolons on end keywords.
- `integrated.html` calls `/api/devices` which does not exist in api.py yet. 
  That endpoint needs to be added in Session 10.
- Token must never live in JS files in production. The HAFetcher class in 
  integrated.html is correctly wired to the backend at port 7777 — keep it 
  that way. The standalone ha-fetcher.js holds the token directly and is for 
  local testing only, never production.
- Wizard capabilities list is still mock data — needs real HA entity attributes.
- Right-click context menu is an alert stub — needs real per-statement implementation.
- filterDevices() doesn't debounce — will hammer the backend on every keystroke.
